HOW TO THINK IN VIM

To be efficient in VIM, learning commands is not enough. To accomplish editing tasks elegantly, you must learn to think in "VIM VERB NOUN MODIFIER" statements. Because VIM is a symbol language with a formal grammatical structure, these VIM idioms describe what you want to do in a somewhat natural way and are the heart of VIM wizardry.
VERBS are the thing you want to do - (c)hange, (d)elete, (y)ank, (v)isual, etc.
MODIFIERS change how nouns are understood - (i)nside, (a)round, (u)ntil, (t)o, (f)ind, etc.
NOUNS are what you are editing - (w)ord, (s)entence, (p)aragraph, (b)lock, etc.

Some examples

diwdelete inside word to delete word under cursor
ciwchange the inner word, respecting delimitation
ci"change inside quotation marks to change string inside next marks
  • ci" change inside quotation marks to change string inside next marks
  • c* change everything from the cursor till the letter x
  • v} visually select (around) this paragraph
  • yi( copy the phrase inside the parentheses
  • dfoo delete from here until the second occurrence of 'foo'
  • fypx then change the occurrence of 'y', move forward 2 words, and change word 'x'
  • h,j,k,l for fine tuning. If you are using them more than a few times, there's probably a more efficient way to move or accomplish what you're doing. Stop and think in sentences!

MISCELLANEOUS COMMANDS

:wq ZZ write (save) and quit file

:q! quit file (discarding any changes)

:r file read file contents into current file
:map x <s> define keystroke x as a command sequence
:unmap x disable abbreviation or mapping
:ab a p abbreviate p as a; when a is typed in insert mode. it expands to full words or phrases (p)
:unab a disable abbreviation for a
:set [all] display options set by user; if all is included, shows default options as well
:set o activate option o
:set o=v assign value v to option o
:set noU deactivate option U
